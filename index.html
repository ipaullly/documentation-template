Introduction to miniapps

What is a mini program

The Mini Program technology is the answer to rapid mobile app development. Mini Programs are sub-applications that run inside the mobile app. You can access various services and features on Mini Programs without the need to install additional applications.

This technology helps you quickly develop and deploy high-quality services and grow the business on mobile apps. It is easily accesible to users and provides a better user experience for mobile users.

With current advancements in the world, a significant percentage of the population can access apps via their smart devices. this frameworks hopes to maximize on this flexibility to propel business interests. With the Mini Programs domiciled within one app (MPESA Super app), users can experience many services i.e insurance, transport, fuel, banking (etc) without installing a multitude of mobile apps on their phones.

The main development language of the Mini Program is JavaScript, and the development experience of the Mini Program is similar to web development. For our developers, the migrating cost of moving from the HTML5 to Mini Program is kept to a minimum.

Setup

Steps Involved in the Development

A mini program is developed in an IDE(mini program studio). This is a desktop application used exclusively for code editing, local debugging, preview on device, and publishing of a mini program.

  1. Log into the mini program developer platform as admin and create a mini program. Submit particulars like name, description, logo image etc.
  2. Assign team members as developers on the mini program created.
  3. Mini program developer develops and debugs the mini program in the IDE.
  4. The mini program developer uploads the code package from IDE to the mini program developer platform, then the mini program admin can apply to publish a new version of the mini program.
  5. Workspace admin reviews and releases the application.
  6. Users can access the mini program in the MPESA app ‘discover’ tab.

Gotcha's

Bus & Train

During the development of the ‘Book A Bus’ and ‘Madaraka Express’ mini programs, there were a number of notable hitches that took some technical maneuverability to surmount.

  1. The Mini Program Framework has no provisions for ensuring the security and integrity of secret keys or tokens used throughout the mini program. This necessitates a config server to serve these variables whenever a user loads into a mini program.
  2. When a user interacts with the mini program on their mobile devices, the display often times out before a user is able to complete their business within the mini program. This tends to reset all selected or input values owing to the stateless nature of the mini program. This necessitates caching of any input the user provides during such a scenario.
  3. Although the MPESA app holds user information, the mini programs currently don’t have capabilities of accessing details like phone number, date of birth or national ID.
  4. Certain APIs used to implement functionalities like the main payment processing call, have a skewed scoping behaviour i.e the use of this.* prefix requires a change to that.*.
  5. Making HTTPS requests requires different considerations for both iOS and Android. Certain URL endpoints cannot be called by passing query parameters, but only by passing a JSON body as the request parameter.
  6. The framework mainly supports javascript for writing page logic, but only allows a limited amount of NPM packages to be installed. This means a majority of validation, form inputs, date inputs and error feedback require the writing of extensive lines of code to implement with vanilla JavaScript.
  7. When publishing a new mini program version on the mini program developer platform it is important to provision payments as a feature and whitelist all domains that are used for HTTPS request calls within the mini program.
  8. The documentation mainly covers the use of components provided by the mini-antui npm library which is deprecated, while current mini programs use the mini-ali-ui package.
